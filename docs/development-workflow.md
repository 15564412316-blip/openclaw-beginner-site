## 二、各阶段详细说明

### 阶段 2：技术架构设计

| 项目 | 说明 |
|------|------|
| **目标** | 设计整体技术架构、目录结构、技术选型 |
| **输入** | PRD 文档 |
| **输出** | 架构设计文档、技术选型说明 |

#### 使用的 ECC 组件

| 组件类型 | 组件名称 | 用途 |
|---------|---------|------|
| **Agent** | `architect` | 系统架构设计决策 |
| **Agent** | `planner` | 功能实现规划 |
| **Command** | `/plan` | 创建实现计划 |
| **Skill** | `api-design` | API 设计规范 |
| **Skill** | `backend-patterns` | 后端架构模式 |
| **Skill** | `frontend-patterns` | 前端架构模式 |

#### 执行命令

```bash
# 创建架构设计
/plan "设计 OpenClaw 新手站的技术架构"

# 或者使用 architect agent
# 委托给架构师代理进行系统设计
```

#### 输出物
- `docs/architecture.md` - 架构设计文档
- `docs/api-design.md` - API 设计文档
- `docs/database-schema.md` - 数据库设计

### 阶段 5：测试验证

| 项目 | 说明 |
|------|------|
| **目标** | 确保代码质量、功能正确 |
| **输入** | 源代码 |
| **输出** | 测试报告 |

#### 使用的 ECC 组件

| 组件类型 | 组件名称 | 用途 |
|---------|---------|------|
| **Agent** | `tdd-guide` | TDD 测试指导 |
| **Agent** | `e2e-runner` | E2E 端到端测试 |
| **Command** | `/tdd` | TDD 工作流 |
| **Command** | `/e2e` | 生成 E2E 测试 |
| **Command** | `/test-coverage` | 测试覆盖率分析 |
| **Command** | `/verify` | 运行验证循环 |
| **Skill** | `tdd-workflow` | TDD 方法论 |

#### 测试类型

| 测试类型 | 说明 | 工具 |
|---------|------|------|
| **单元测试** | 测试单个函数/组件 | Jest/Vitest |
| **集成测试** | 测试模块间交互 | Testing Library |
| **E2E 测试** | 测试完整用户流程 | Playwright |
| **类型检查** | TypeScript 类型验证 | `tsc --noEmit` |
| **Lint 检查** | 代码规范检查 | ESLint |

#### 执行命令

```bash
# 运行类型检查
npm run lint

# 运行构建（验证编译）
npm run build

# 运行 E2E 测试
/e2e
```

| **输入** | 测试通过的代码 |

| 组件类型 | 组件名称 | 用途 |

| **Command** | `/build-fix` | 修复构建错误 |

#### 部署步骤

1. **代码推送到 GitHub**
   ```bash
   git add .
   git commit -m "feat: 完成首页开发"
   git push origin main
   ```

2. **Cloudflare Pages 配置**
   - 连接 GitHub 仓库
   - 配置构建命令: `npm run build`
   - 配置输出目录: `.next`
   - 配置环境变量

3. **域名绑定（可选）**

## 三、开发流程执行顺序

```
Week 1: 基础搭建
├── Day 1-2: 技术架构设计 (architect + planner)
├── Day 3-4: 首页开发 (frontend-patterns)
└── Day 5: 首页测试 + 审核 (code-review + verify)

Week 2: 核心页面
├── Day 1-2: 安装路径选择器
├── Day 3: API Key 接入中心
├── Day 4: 首次任务模板
└── Day 5: 测试 + 审核

Week 3: 完善上线
├── Day 1-2: FAQ + 留资页面
├── Day 3: 安全审查 + 修复
├── Day 4: 部署配置
└── Day 5: 上线验证
```

## 四、ECC 组件速查表

### Agents（代理）- 负责人

|-------|------|-----------|
| `architect` | 架构师 | 设计系统架构、技术选型 |
| `planner` | 规划师 | 规划功能实现步骤 |
| `tdd-guide` | 测试专家 | 指导测试驱动开发 |
| `code-reviewer` | 代码审核员 | 审查代码质量 |
| `security-reviewer` | 安全专家 | 安全漏洞检查 |
| `build-error-resolver` | 构建专家 | 解决构建错误 |
| `e2e-runner` | 测试工程师 | 运行端到端测试 |
| `refactor-cleaner` | 重构专家 | 清理死代码 |
| `doc-updater` | 文档专家 | 更新文档 |

### Commands（命令）- 快捷操作

| Command | 作用 | 使用场景 |

| `/plan` | 创建实现计划 | 开始新功能前 |

| `/verify` | 运行验证循环 | 验证功能时 |
| `/refactor-clean` | 清理死代码 | 代码整理时 |
